🧠 PERFECTFIT — Full System Architecture & Statement of Work (SOW)
1. System Overview

PerfectFit is an AI-driven personalized shopping and tailoring marketplace for men’s fashion.
It merges:

Measurement-based fit matching

Style recommendations based on age, lifestyle, and hobbies

Budget optimization

Quick Buy retail purchasing

Custom Clothing Network, where users can receive bespoke recommendations and send requests to tailors.

2. Platform Components
Layer	Core Purpose	Key Tech
Frontend	Mobile/web app (Flutter) for users & makers	Flutter, Dart
Backend	Business logic, AI parsing, APIs, matchmaking	Python (FastAPI/Flask)
Database	User, Maker, Product, Order, RFQ data	PostgreSQL / Firebase

Integrations	Retail APIs, Payment gateways	Amazon, Shopify, Stripe, PayPal
Hosting	Dev on Replit, Prod on AWS/Firebase	AWS EC2/S3, Firebase Hosting

5. Database Schema (Simplified)

Core Tables:

users (id, name, email, role)

measurements (user_id, chest, waist, etc.)

user_profiles (age, style_tags, hobbies, budget)

products (id, brand, price, fit_score, style_match)

makers (id, specialties, style_tags, price_range, location)

custom_requests (user_id, item_type, budget, notes, status)

quotes (request_id, maker_id, price, lead_time, message)

orders (retail/bespoke, user_id, total, status)

payments (order_id, type, provider, amount, status)

6. Core Backend Services
Service	Purpose
ai_parser.py	Converts freehand user text into structured data (age, lifestyle, style, budget).
recommendation_service.py	Ranks retail items by Fit + Style + Budget.
maker_match_service.py	Matches users to tailors based on style tags, pricing, and garment type.
rfq_service.py	Handles custom request creation and quote distribution.
affiliate_service.py	Creates Quick Buy affiliate checkout URLs.
payments_service.py	Manages Stripe/PayPal deposits and balances.
7. User Flow Scenarios
🔹 Scenario 1: Retail Quick Buy

User enters measurements and preferences.

System fetches matching products (e.g., Amazon, ASOS).

Returns list ranked by Fit + Style + Budget.

User clicks “Quick Buy” → redirected to checkout page or in-app Stripe session.

🔹 Scenario 2: Bespoke Custom Request

User selects “Request Custom Suit.”

AI parses user style & budget.

Makers matching those parameters receive RFQ.

Makers submit quotes (price + delivery + fabric).

User selects one, pays deposit → order created.

Tailor updates production status → final payment upon completion.

8. Scoring Logic

Retail Recommendation

Score = 0.5 * FitScore + 0.3 * StyleMatch + 0.2 * BudgetFit


Maker Match

MakerScore = 0.4 * StyleOverlap + 0.3 * BudgetCompatibility + 0.2 * Expertise + 0.1 * LeadTime

14. README (Developer Summary)
🧩 Quick Start on Replit

Clone repo or import ZIP.

In /backend, run:

pip install -r requirements.txt
python app.py


In /frontend, run:

flutter run -d chrome


Add .env with:

OPENAI_API_KEY=
STRIPE_SECRET_KEY=
POSTGRES_URL=


Open Swagger at localhost:8080/docs.
